# Contributing to DevStack

First off, thank you for considering contributing to DevStack! It's people like you that make DevStack such a great tool.

## How Can I Contribute?

### Reporting Bugs
- Use the GitHub issue tracker to report bugs.
- Describe the expected behavior and the actual behavior.
- Include steps to reproduce the issue.

### Suggesting Enhancements
- Open a GitHub issue with the tag "enhancement".
- Explain why this feature would be useful.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. Ensure the test suite passes.
4. Issue that Pull Request!

## Development Process

- **Branching:** Use descriptive branch names (e.g., `feat/add-github-auth`).
- **Commits:** Follow conventional commits (e.g., `feat: ...`, `fix: ...`, `docs: ...`).
- **Code Style:** We use Prettier and ESLint. Please ensure your code follows the project's style.

## Community

Join our discussions on the GitHub repository for any questions or architectural debates!
